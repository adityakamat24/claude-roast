# claude-roast

**A zero-token aura meter in the Claude Code statusline. Trivial prompts cost you. Real work earns it back.**

```
😎 aura 320  [▓▓▓▓▓░░░░░]  locked in   last: shipped 6 edits +180
💀 aura -980 [░░░░░░░░░░]  smooth brain  last: mkdir (alone) -150
```

---

## What it is

claude-roast runs a live "aura score" in your statusline that measures whether you are using Claude Code well or wasting a frontier model on busywork. Three shell hooks act as sensors: one fires when you submit a prompt, one logs every tool Claude uses during the turn, and one commits the net score when the turn ends. A statusline renderer reads the state file and draws the meter on every update.

The model is **never invoked to score anything**. All scoring is pure shell heuristics running outside Claude's context window, adding nothing to it. **Zero tokens by default.**

The meter is two-way. You can claw aura back. A one-way death spiral would be boring by 4 PM.

## Turn on the meter (required second step)

> **Important:** A plugin cannot set the main `statusLine` directly — Claude Code only allows plugins to write `agent`/`subagentStatusLine`, not `statusLine`. You must wire the meter yourself. This takes one command or one line in settings.

**Easiest** — run after installing the plugin:

```
/claude-roast install
```

**Or manually** — add to `~/.claude/settings.json`:

```json
"statusLine": { "type": "command", "command": "bash \"$HOME/.claude/aura/statusline.sh\"" }
```

The meter state lives in `~/.claude/aura/` and survives plugin updates. It is seeded automatically on SessionStart.

---

## Compose with an existing statusline

If you already have a `statusLine` configured, only one command can drive it at a time. Run `/claude-roast install` anyway: it detects the existing value, backs up `~/.claude/settings.json`, and generates `~/.claude/aura/statusline-wrapper.sh`. That wrapper runs your existing statusline command and appends the aura segment via `statusline.sh --segment`. Point `statusLine` at the wrapper.

Back up `~/.claude/settings.json` before making any manual changes.

## Commands

| Command | What it does |
|---------|--------------|
| `/claude-roast` | Show current aura score, rank, and recent history |
| `/claude-roast shame` | Print the hall of shame — your five worst turns |
| `/claude-roast roast` | On-demand AI roast of your recent form (see note below) |
| `/claude-roast reset` | Reset the meter — asks for confirmation first |
| `/claude-roast install` | Wire `statusLine` to the aura meter |

**On `/claude-roast roast`:** this is the only model-powered feature. It reads your state and recent history and produces a savage verdict, generated by your current Claude session using your own authentication. It costs only your normal usage and fires only when you explicitly invoke it. The model cannot auto-fire this per turn — doing so would silently drain your quota and get the plugin uninstalled.

## Scoring at a glance

**Drains aura**

| Behaviour | Delta |
|-----------|------:|
| One or two-word greeting / ack (`hi`, `yo`, `thanks`, `ok`, …) | −40 |
| Folder or file-only intent (`mkdir`, `create a folder`, `rename this file`, …) | −60 |
| Near-duplicate of your previous prompt | −50 |
| ALL CAPS (≥80% uppercase in a message ≥6 chars) | −15 |
| Begging (`pls pls`, `please please`) | −15 |
| **The centrepiece: entire turn was one trivial shell command** (`mkdir`, `ls`, `touch`, `cat`, `pwd`, …) | **−150** |
| Turn was a single Read and nothing else | −40 |

**Farms aura**

| Behaviour | Delta |
|-----------|------:|
| Long, structured prompt (≥240 chars or ≥3 sentences) | +50 base |
| Per additional 100 characters | +15 each |
| Prompt contains a fenced code block | +60 bonus |
| Prompt gains from detail are capped at | +150 total |
| Multi-file edits (Edit / Write / MultiEdit) | +30 per file, cap +180 |
| Turn ran tests or a build (`pytest`, `cargo test`, `npm test`, etc.) | +80 |
| Busy turn (6+ tool calls) | +30 bonus |

Per-turn swings are clamped to [−300, +300].

Everything is tunable: numbers and regex patterns live in `~/.claude/aura/config.json` (your copy, seeded from the plugin's `config.json`, wins on every key). Verdict lines live in `verdicts/*.txt`. Rank tiers live in `ranks.tsv`.

## Natural rating with Claude (optional, off by default)

By default claude-roast is **pure heuristics — zero tokens, no model calls.** But heuristics don't *understand* a turn; they pattern-match. If you'd rather have Claude **actually judge** each turn (catching what regex can't — e.g. a substantive pure-chat question that heuristics score a flat `neutral 0`), turn on the model judge in `~/.claude/aura/config.json`:

```json
"judge_mode": "gated"
```

- **`off`** (default) — heuristics only. Zero tokens.
- **`self`** ⭐ recommended — **the session model itself** (e.g. Opus, the one you're already talking to) rates each turn. A `UserPromptSubmit` hook quietly asks it to end its reply with a hidden `<!--aura: N | verdict -->` marker; the `Stop` hook extracts that from the transcript. **No separate call, no API key, near-instant** (the verdict is already in the reply the moment the turn ends), and the **funniest** option — it's your main model, and it just *did* the turn so it has full context, not a summary. ~40 tokens folded into the turn you were already paying for.
- **`gated`** — a *separate, cheaper* model judges only the turns heuristics can't (pure chat, conflicting signals). Runs async. ~80–90% of turns stay free.
- **`always`** — a *separate* model rates every turn, async.

`self` and `gated`/`always` both **fall back to the heuristic** on any miss (the model skips the marker, you're offline, a timeout), so they can never stall or break a turn. `gated`/`always` run **async** (the heuristic shows instantly, the model refines a beat later); `self` is synchronous-instant (nothing to wait for — the verdict rode along in the reply). Model-rated turns are tagged `model`/`claude` in `/claude-roast` and the hall of shame.

**`gated`/`always` only — which separate model / how lean?** Controlled by `judge_cmd` (`self` and `off` ignore this):

| `judge_cmd` | What it uses | Setup | Cost / speed |
|---|---|---|---|
| `auto` (default) | direct Anthropic API (`rate-api.sh`) **if `ANTHROPIC_API_KEY` is set**, else `claude -p` | none, or a key | API: ~170 tokens, ~1–2s · `claude -p`: heavier, ~20–30s |
| `claude` | `claude -p` on your Claude Code login | none | no key, but loads the full agent each call (slower, more tokens) |
| *custom cmd* | anything speaking `cmd -p --model <m> "<prompt>"` | yours | e.g. a local **Ollama** wrapper → zero API tokens |

**Recommended:** `judge_mode: self` — funniest, near-instant, no key, no separate call. Use `gated` + `export ANTHROPIC_API_KEY=…` only if you specifically want the rating done by a *separate* model out-of-band. The `self`-mode instruction lives in `score-prompt.sh`; the separate-model rubric is `judge-prompt.txt`. Quick toggle without editing config: `export AURA_JUDGE_MODE=self` (or `off`/`gated`/`always`).

## Write your own verdict pack

Verdict packs are plain-text files in `verdicts/<pack>.txt`. One verdict per line. Lines starting with `#` are comments and are skipped. In `shipped_code` verdicts, `{n}` is replaced by the actual edit count for that turn.

Built-in packs:

| Pack | When it fires |
|------|---------------|
| `trivial_solo` | Entire turn was one trivial shell command |
| `just_a_read` | Turn was a single Read and nothing else |
| `one_word` | One or two-word greeting or acknowledgement |
| `folder_only` | Folder / file-only intent in the prompt |
| `dupe_prompt` | Near-duplicate of the previous prompt |
| `shouting` | All caps |
| `begging` | Repeated pleading |
| `detailed_prompt` | Long, structured, specific prompt |
| `shipped_code` | Multiple edits across files (supports `{n}`) |
| `shipped_tests` | Turn ran tests or a build |
| `neutral` | Pure chat, no tools used |

Keep verdicts savage but fair. Roast the *request*, never the person. PRs with new packs are welcome — drop the file in `verdicts/` and it is picked up automatically.

## Windows

Scripts are pure bash, run via Git Bash — the same shell Claude Code uses for hooks and the statusline on Windows. Use forward slashes and `$HOME` throughout. PowerShell is not involved.

## Compatibility note (2026-06)

Built against `code.claude.com/docs`. One notable deviation from the original design brief: plugins cannot auto-wire `statusLine` (only `agent`/`subagentStatusLine` are plugin-writable), so the meter must be connected via `/claude-roast install` or one line in user settings. The `UserPromptSubmit` hook delivers the prompt text in the `user_message` field (with fallback to `prompt`).

## Privacy and safety

- No tokens are spent by default. Nothing leaves your machine.
- No bundled API key or credentials of any kind.
- Nothing hosted. The plugin is a static git repo.
- Hooks exit fast. The meter never blocks or breaks a session.

---

## Uninstall

1. Remove the `statusLine` entry from `~/.claude/settings.json` (or restore the backup that `/claude-roast install` created).
2. Run `/plugin uninstall claude-roast`.
3. Optionally delete `~/.claude/aura/` to wipe all meter state.
